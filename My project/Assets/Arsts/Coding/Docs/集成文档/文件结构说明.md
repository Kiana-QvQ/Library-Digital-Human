# ASR&TTS 文件结构说明

## 目录结构

```
Coding/ASR&TTS/
├── Coze/                    # Coze语音服务
│   ├── CozeStreamTTS.cs              # WebSocket流式TTS（可独立使用）
│   ├── CozeTextToSpeech.cs           # TTS封装（使用CozeStreamTTS）
│   ├── CozeVoiceClone.cs             # 语音克隆（可独立使用）
│   ├── CozeVoiceAPI.cs               # 音色列表查询（可独立使用）
│   ├── CozeStreamTTSExample.cs       # WebSocket使用示例
│   ├── CozeVoiceCloneExample.cs      # 语音克隆使用示例
│   └── CozeVoiceAPIExample.cs        # 音色查询使用示例
│
├── Doubao/                  # Doubao（火山引擎）语音服务
│   ├── DoubaoStreamTTS.cs            # WebSocket流式TTS V1（可独立使用）
│   ├── DoubaoStreamTTSV3.cs          # WebSocket流式TTS V3（可独立使用）
│   ├── DoubaoTextToSpeech.cs         # TTS封装（使用DoubaoStreamTTS）
│   ├── DoubaoVoiceClone.cs           # 语音克隆（可独立使用）
│   ├── DoubaoStreamTTSV3_README.md  # V3使用说明
│   └── DoubaoVoiceCloneExample.cs    # 语音克隆使用示例
│
├── Baidu/                   # 百度语音服务
│   └── ...
│
├── TTS.cs                   # TTS基类
└── STT.cs                   # STT基类
```

## 功能独立性

### Coze服务

#### 1. WebSocket流式TTS（CozeStreamTTS）
- **位置**：`Coding/ASR&TTS/Coze/CozeStreamTTS.cs`
- **独立性**：✅ 可独立使用
- **依赖**：仅需API Key
- **音色支持**：默认不使用音色（`useVoiceId = false`）
- **使用方式**：
  ```csharp
  CozeStreamTTS streamTTS = gameObject.AddComponent<CozeStreamTTS>();
  streamTTS.SetAPIKey("your_api_key");
  streamTTS.Connect();
  streamTTS.SynthesizeText("测试文本");
  ```

#### 2. 语音克隆（CozeVoiceClone）
- **位置**：`Coding/ASR&TTS/Coze/CozeVoiceClone.cs`
- **独立性**：✅ 可独立使用
- **依赖**：仅需API Key
- **使用方式**：
  ```csharp
  CozeVoiceClone voiceClone = gameObject.AddComponent<CozeVoiceClone>();
  voiceClone.SetAPIKey("your_api_key");
  voiceClone.CloneVoiceFromFile("audio.wav", "音色名称");
  ```

### Doubao服务

#### 1. WebSocket流式TTS（DoubaoStreamTTS / DoubaoStreamTTSV3）
- **位置**：
  - `Coding/ASR&TTS/Doubao/DoubaoStreamTTS.cs`（V1版本）
  - `Coding/ASR&TTS/Doubao/DoubaoStreamTTSV3.cs`（V3版本）
- **独立性**：✅ 可独立使用
- **依赖**：仅需AppID和Token
- **音色支持**：默认不使用音色（`useSpeakerId = false`）
- **使用方式**：
  ```csharp
  DoubaoStreamTTS streamTTS = gameObject.AddComponent<DoubaoStreamTTS>();
  streamTTS.SetCredentials("appId", "token");
  streamTTS.Connect();
  streamTTS.SynthesizeText("测试文本");
  
  // 或使用V3版本
  DoubaoStreamTTSV3 streamTTSV3 = gameObject.AddComponent<DoubaoStreamTTSV3>();
  streamTTSV3.SetCredentials("appId", "token", "seed-tts-1.0");
  streamTTSV3.Connect();
  streamTTSV3.SynthesizeText("测试文本");
  ```

#### 2. 语音克隆（DoubaoVoiceClone）
- **位置**：`Coding/ASR&TTS/Doubao/DoubaoVoiceClone.cs`
- **独立性**：✅ 可独立使用
- **依赖**：仅需AppID和Token
- **使用方式**：
  ```csharp
  DoubaoVoiceClone voiceClone = gameObject.AddComponent<DoubaoVoiceClone>();
  voiceClone.SetCredentials("appId", "token");
  voiceClone.CloneVoiceFromFile("audio.wav", "S_MyVoice");
  ```

## 组合使用

### 场景1：仅使用WebSocket TTS（不使用音色）
```csharp
// Coze
CozeStreamTTS cozeTTS = gameObject.AddComponent<CozeStreamTTS>();
cozeTTS.SetAPIKey("api_key");
cozeTTS.useVoiceId = false; // 默认值，不使用音色
cozeTTS.Connect();

// Doubao
DoubaoStreamTTS doubaoTTS = gameObject.AddComponent<DoubaoStreamTTS>();
doubaoTTS.SetCredentials("appId", "token");
doubaoTTS.SetUseSpeakerId(false); // 默认值，不使用音色
doubaoTTS.Connect();
```

### 场景2：仅使用语音克隆
```csharp
// Coze（注意：由于权限问题，可能无法使用）
CozeVoiceClone cozeClone = gameObject.AddComponent<CozeVoiceClone>();
cozeClone.SetAPIKey("api_key");
cozeClone.CloneVoiceFromFile("audio.wav", "音色名称");

// Doubao（推荐）
DoubaoVoiceClone doubaoClone = gameObject.AddComponent<DoubaoVoiceClone>();
doubaoClone.SetCredentials("appId", "token");
doubaoClone.CloneVoiceFromFile("audio.wav", "S_MyVoice");
```

### 场景3：WebSocket TTS + 语音克隆
```csharp
// 1. 先进行语音克隆
DoubaoVoiceClone voiceClone = gameObject.AddComponent<DoubaoVoiceClone>();
voiceClone.SetCredentials("appId", "token");
voiceClone.CloneVoiceFromFile("audio.wav", "S_MyVoice");

// 2. 克隆成功后，使用WebSocket TTS并启用音色
DoubaoStreamTTS streamTTS = gameObject.AddComponent<DoubaoStreamTTS>();
streamTTS.SetCredentials("appId", "token");
streamTTS.SetSpeakerId("S_MyVoice"); // 自动启用音色使用
streamTTS.Connect();
streamTTS.SynthesizeText("使用克隆音色合成文本");
```

## 与API管理器集成（可选）

虽然所有组件都可以独立使用，但通过API管理器可以统一管理凭证和配置：

### CozeAPIManager
- **位置**：`Coding/Main/CozeAPIManager.cs`
- **功能**：统一管理Coze API Key和音色ID
- **使用**：可选，组件可以独立工作

### DoubaoAPIManager
- **位置**：`Coding/Main/DoubaoAPIManager.cs`
- **功能**：统一管理Doubao凭证、音色ID和语音克隆开关
- **使用**：可选，组件可以独立工作

## 默认设置

### WebSocket组件
- **CozeStreamTTS**：`useVoiceId = false`（默认不使用音色）
- **DoubaoStreamTTS**：`useSpeakerId = false`（默认不使用音色）
- **DoubaoStreamTTSV3**：`useSpeakerId = false`（默认不使用音色）

### 语音克隆
- **CozeVoiceClone**：需要API Key，但可能因权限问题无法使用
- **DoubaoVoiceClone**：需要AppID和Token，推荐使用

## 注意事项

1. **独立性**：所有WebSocket和语音克隆组件都可以独立使用，不依赖API管理器
2. **音色默认关闭**：所有WebSocket组件的音色参数默认不使用，需要手动启用
3. **组合使用**：可以先进行语音克隆，然后在WebSocket中启用音色使用
4. **API管理器**：可选，用于统一管理，但不强制使用

## 文件位置说明

- **ASR&TTS文件夹**：所有语音相关的组件都在这里
- **Main文件夹**：API管理器在主控制器文件夹中，用于统一管理
- **Chat文件夹**：聊天相关的组件（如CozeClient）

这种结构确保了：
- 功能模块清晰分离
- 组件可以独立使用
- 可以灵活组合使用
- 便于维护和扩展

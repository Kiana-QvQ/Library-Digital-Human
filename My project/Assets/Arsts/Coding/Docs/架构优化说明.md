# 代码架构优化说明

## 优化内容

### 1. 修复的问题

#### 类名一致性
- ✅ 修复了 `VoiceControlManager.cs` 中类名与文件名不一致的问题
  - 原类名：`VoiceControlScript`
  - 新类名：`VoiceControlManager`

#### 注释乱码修复
- ✅ 修复了 `STT.cs` 和 `TTS.cs` 中的中文注释乱码问题
  - 所有注释已转换为正确的中文编码

### 2. 新增的接口定义

为了统一不同提供者的接口，新增了以下接口：

#### ITTSProvider（TTS提供者接口）
- 位置：`Coding/ASR&TTS/ITTSProvider.cs`
- 用途：统一TTS提供者的接口，便于管理和切换
- 属性：
  - `ProviderName`: 提供者名称
  - `IsAvailable`: 是否可用
  - `Priority`: 优先级
- 方法：
  - `Speak(string text, Action<AudioClip, string> callback)`: 文本转语音

#### ISTTProvider（STT提供者接口）
- 位置：`Coding/ASR&TTS/ISTTProvider.cs`
- 用途：统一STT提供者的接口，便于管理和切换
- 属性：同ITTSProvider
- 方法：
  - `SpeechToText(AudioClip clip, Action<string> callback)`: 语音转文本（AudioClip）
  - `SpeechToText(byte[] audioData, Action<string> callback)`: 语音转文本（字节数组）

#### ILLMProvider（LLM提供者接口）
- 位置：`Coding/Chat/ILLMProvider.cs`
- 用途：统一LLM提供者的接口，便于管理和切换
- 属性：同ITTSProvider
- 方法：
  - `PostMessage(string message, Action<string> callback)`: 发送消息

### 3. 新增的基类

#### BaseAPIManager（API管理器基类）
- 位置：`Coding/Main/BaseAPIManager.cs`
- 用途：提供统一的API管理器模式，减少代码重复
- 功能：
  - 统一的调试日志方法（`LogDebug`, `LogWarning`, `LogError`）
  - 抽象方法：
    - `InitializeComponents()`: 初始化组件引用
    - `SyncToAllComponents()`: 同步配置到所有组件

#### 继承关系
- `CozeAPIManager` 现在继承自 `BaseAPIManager`
- `DoubaoAPIManager` 现在继承自 `BaseAPIManager`

### 4. 事件总线系统

#### VoiceEventBus（语音系统事件总线）
- 位置：`Coding/Main/VoiceEventBus.cs`
- 用途：解耦各个模块之间的通信，通过事件系统进行消息传递
- 事件类型：
  - `OnSpeechRecognized`: 语音识别完成
  - `OnLLMResponse`: LLM回复
  - `OnTTSComplete`: TTS合成完成
  - `OnTTSError`: TTS错误
  - `OnAnimationStateChanged`: 动画状态变化
  - `OnSystemEvent`: 系统事件（用于日志记录等）

## 使用说明

### 接口的使用（可选）

现有的实现类（如 `BaiduTextToSpeech`, `CozeTextToSpeech` 等）可以逐步实现这些接口，但**不是必须的**。这些接口为未来的重构提供了基础，不会影响现有代码的运行。

### 事件总线的使用（可选）

如果需要使用事件总线来解耦代码，可以这样使用：

```csharp
// 订阅事件
VoiceEventBus.OnLLMResponse += OnLLMResponseReceived;

// 触发事件
VoiceEventBus.EmitLLMResponse(responseText);
```

**注意**：事件总线是可选的，现有代码可以继续使用原有的回调方式。

### API管理器的使用

API管理器现在继承自 `BaseAPIManager`，但所有原有的方法和属性都保持不变，可以继续正常使用。

## 注意事项

1. **向后兼容**：所有优化都保持了向后兼容，不会影响现有代码的运行
2. **渐进式重构**：接口和事件总线是可选的，可以逐步采用
3. **不改变逻辑**：所有优化都没有改变原有的业务逻辑和处理流程

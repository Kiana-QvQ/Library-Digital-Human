# 屏幕适配管理器使用说明

## 功能概述

`ScreenAdaptationManager` 是一个通用的屏幕适配脚本，可以自动调整场景中的背景、画布等对象及其子对象的缩放，以适应不同的屏幕分辨率，同时保持对象的位置不变。

## 主要特性

- ✅ 自动适配不同屏幕分辨率
- ✅ 保持对象位置不变（只调整缩放）
- ✅ 支持Canvas、GameObject、Transform等多种对象类型
- ✅ 自动适配所有子对象
- ✅ 运行时动态监听分辨率变化
- ✅ 支持多种适配模式
- ✅ 可设置缩放限制范围
- ✅ 提供调试信息

## 使用方法

### 1. 添加脚本到场景

1. 在场景中创建一个空的GameObject（例如命名为"ScreenAdaptationManager"）
2. 将 `ScreenAdaptationManager.cs` 脚本添加到该GameObject上

### 2. 配置参考分辨率

在Inspector中设置：
- **Reference Resolution**：参考分辨率（设计时的目标分辨率）
  - 例如：1920x1080（16:9）、1280x720（16:9）、1024x768（4:3）等

### 3. 选择适配模式

- **Match Width**：匹配宽度（以宽度为准）
- **Match Height**：匹配高度（以高度为准）
- **Match Width Or Height**：匹配宽度或高度（根据matchWidthOrHeight参数）
- **Expand**：扩展（取较大的缩放比例，可能超出屏幕）
- **Shrink**：收缩（取较小的缩放比例，确保不超出屏幕）

### 4. 添加需要适配的对象

在Inspector中，将需要适配的对象拖拽到对应的列表中：
- **Canvas List**：Canvas对象列表
- **Game Object List**：普通GameObject列表
- **Transform List**：Transform列表

**注意**：脚本会自动适配这些对象及其所有子对象。

### 5. 配置适配选项

- **Adapt On Start**：是否在Start时自动适配（推荐开启）
- **Adapt On Resolution Change**：是否监听屏幕分辨率变化（运行时动态适配）
- **Check Interval**：分辨率变化检测间隔（秒）

### 6. 设置缩放限制（可选）

- **Min Scale**：最小缩放比例（防止对象过小）
- **Max Scale**：最大缩放比例（防止对象过大）

## 代码示例

### 运行时添加对象

```csharp
// 获取ScreenAdaptationManager实例
ScreenAdaptationManager manager = FindObjectOfType<ScreenAdaptationManager>();

// 添加Canvas
Canvas myCanvas = GetComponent<Canvas>();
manager.AddCanvas(myCanvas);

// 添加GameObject
GameObject background = GameObject.Find("Background");
manager.AddGameObject(background);

// 添加Transform
Transform uiRoot = transform.Find("UIRoot");
manager.AddTransform(uiRoot);
```

### 手动触发适配

```csharp
ScreenAdaptationManager manager = FindObjectOfType<ScreenAdaptationManager>();
manager.ManualAdapt(); // 手动触发适配
```

### 移除对象

```csharp
// 移除Canvas
manager.RemoveCanvas(myCanvas);

// 移除GameObject
manager.RemoveGameObject(background);

// 移除Transform
manager.RemoveTransform(uiRoot);
```

### 重置到原始缩放

```csharp
manager.ResetToOriginalScale(); // 重置所有对象到原始缩放
```

## 适配模式详解

### Match Width（匹配宽度）
- 以屏幕宽度为基准进行缩放
- 适用于横向内容较多的场景
- 公式：`scale = 屏幕宽度 / 参考宽度`

### Match Height（匹配高度）
- 以屏幕高度为基准进行缩放
- 适用于纵向内容较多的场景
- 公式：`scale = 屏幕高度 / 参考高度`

### Match Width Or Height（匹配宽度或高度）
- 根据 `matchWidthOrHeight` 参数在宽度和高度之间插值
- `0` = 完全匹配宽度
- `1` = 完全匹配高度
- `0.5` = 两者兼顾
- 公式：`scale = Lerp(scaleX, scaleY, matchWidthOrHeight)`

### Expand（扩展）
- 取宽度和高度缩放比例中的较大值
- 确保内容完全显示，但可能超出屏幕
- 公式：`scale = Max(scaleX, scaleY)`

### Shrink（收缩）
- 取宽度和高度缩放比例中的较小值
- 确保内容不超出屏幕，但可能留有空白
- 公式：`scale = Min(scaleX, scaleY)`

## 注意事项

1. **原始缩放值**：脚本会在Awake时保存所有对象的原始缩放值，确保基于原始值进行缩放

2. **位置不变**：脚本只调整 `localScale`，不改变 `position`，确保对象位置不变

3. **子对象自动适配**：添加到列表的对象的所有子对象都会自动适配

4. **运行时添加**：可以在运行时动态添加或移除对象，脚本会自动处理

5. **性能考虑**：
   - 分辨率检查间隔不要设置太小（建议0.5秒以上）
   - 如果不需要运行时动态适配，可以关闭 `Adapt On Resolution Change`

6. **Canvas设置**：如果适配Canvas，建议Canvas的Render Mode设置为"Screen Space - Overlay"或"Screen Space - Camera"

## 调试

开启 **Show Debug Info** 可以在Console中查看适配信息：
- 当前屏幕分辨率
- 参考分辨率
- 计算出的缩放比例

## 常见问题

### Q: 为什么对象位置改变了？
A: 确保脚本只调整了 `localScale`，没有修改 `position`。如果位置改变，可能是对象本身有动画或其他脚本在修改位置。

### Q: 适配后对象太小/太大？
A: 调整 `Min Scale` 和 `Max Scale` 参数，限制缩放范围。

### Q: 某些子对象没有适配？
A: 确保父对象在适配列表中，脚本会自动适配所有子对象。如果子对象在运行时动态创建，需要手动调用 `AddTransform()` 添加。

### Q: 如何恢复原始大小？
A: 调用 `ResetToOriginalScale()` 方法，或移除对象后重新添加。

## 最佳实践

1. **设计时确定参考分辨率**：根据目标设备确定参考分辨率（如1920x1080）

2. **选择合适的适配模式**：
   - UI界面：推荐 `Match Width Or Height`（matchWidthOrHeight = 0.5）
   - 背景图片：推荐 `Shrink`（确保不超出屏幕）
   - 游戏内容：根据内容特点选择

3. **设置合理的缩放限制**：防止极端分辨率下对象过小或过大

4. **测试多种分辨率**：在Unity Editor中测试不同分辨率，确保适配效果符合预期

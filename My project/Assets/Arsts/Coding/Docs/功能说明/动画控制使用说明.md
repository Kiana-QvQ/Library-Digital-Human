# 动画控制使用说明

## 功能概述

已为 `VoiceControlManager.cs` 添加了动画状态自动控制功能，可以根据对话流程自动切换角色的动画状态。

## 动画状态映射

根据您的 Animator Controller 配置，动画状态映射如下：

- **state = 0**: 待机（默认状态）
- **state = 1**: 思考
- **state = 2**: 回答动作

## 动画状态转换流程

1. **收到用户请求** → 切换到"思考"状态（state = 1）
   - 触发位置：`SendMessageToAI()` 方法中

2. **开始生成文字和语音** → 切换到"回答动作"状态（state = 2）
   - 触发位置：`OnAgentResponseReceived()` 方法中，收到AI回复时

3. **回答完毕** → 切换回"待机"状态（state = 0）
   - 触发位置：TTS播放完成后（`ResetWakeWordStateAfterDelay()` 或 `CleanupTTSAudioClipAfterDelay()`）

## Unity Inspector 配置

在 Unity Editor 中配置 `VoiceControlManager` 组件时，需要设置：

1. **Character Animator**：将角色的 Animator 组件拖拽到此字段
   - 如果未手动设置，系统会自动尝试从当前 GameObject 或子对象中查找 Animator 组件

## 代码说明

### 新增的公共字段

```csharp
[Header("动画控制")]
public Animator characterAnimator;  // 角色Animator组件
```

### 新增的方法

- `SetAnimationState(int state)`: 设置动画状态
- `GetStateName(int state)`: 获取状态名称（用于日志）

## 注意事项

1. **Animator 参数名称**：代码中使用的参数名称为 `"state"`，请确保您的 Animator Controller 中有一个名为 `state` 的 Int 类型参数。

2. **自动查找**：如果未在 Inspector 中手动设置 `characterAnimator`，系统会在初始化时自动尝试查找：
   - 首先从当前 GameObject 查找
   - 如果未找到，则从子对象中查找

3. **错误处理**：如果 Animator 组件未设置或未找到，系统会在控制台输出警告信息，但不会影响其他功能的正常使用。

4. **日志记录**：所有动画状态切换都会记录到系统日志中，方便调试和追踪。

## 测试建议

1. 在 Unity Editor 中运行场景
2. 确保角色的 Animator Controller 已正确配置
3. 在 `VoiceControlManager` 组件中设置 `Character Animator` 字段
4. 发送一条消息给AI，观察动画状态是否正确切换：
   - 发送消息后 → 应该切换到"思考"状态
   - 收到AI回复后 → 应该切换到"回答动作"状态
   - TTS播放完成后 → 应该切换回"待机"状态

## 故障排除

如果动画状态没有正确切换，请检查：

1. **Animator 组件**：确认 `characterAnimator` 字段已正确设置
2. **参数名称**：确认 Animator Controller 中有名为 `state` 的 Int 类型参数
3. **参数值映射**：确认 Animator Controller 中的转换条件正确配置：
   - state == 0 → 待机
   - state == 1 → 思考
   - state == 2 → 回答动作
4. **控制台日志**：查看 Unity Console 中的日志信息，确认动画状态切换是否被正确调用

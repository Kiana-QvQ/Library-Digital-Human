# 前后端分离架构设计文档

## 概述

本文档描述如何将Unity前端（Coding）与Python后端（digital_human_backend）进行整合，实现前后端分离的架构设计。

## 架构目标

1. **统一接口层**：前端通过统一接口调用，不关心底层是API还是本地服务
2. **适配器模式**：使用适配器封装不同数据源
3. **配置驱动**：运行时切换模式
4. **标准通信**：HTTP REST + WebSocket

## 服务模式

### ServiceMode枚举

```csharp
public enum ServiceMode
{
    API,      // 使用外部API（Coze/Doubao/Baidu）
    Local     // 使用本地后端服务
}
```

## 前端实现方案

### 1. ServiceProvider（统一服务提供者）

位置：`Coding/Main/ServiceProvider.cs`

功能：
- 管理服务模式（API/Local）
- 提供统一的服务接口
- 初始化对应的适配器

### 2. 本地服务适配器

#### LocalLLMProvider
- 位置：`Coding/Main/LocalLLMProvider.cs`
- 功能：通过HTTP/WebSocket调用后端LLM服务
- 接口：`/api/chat/` 和 `/ws/chat/{session_id}`

#### LocalTTSProvider
- 位置：`Coding/Main/LocalTTSProvider.cs`
- 功能：通过HTTP调用后端TTS服务
- 接口：`/api/tts/synthesize`

#### LocalSTTProvider
- 位置：`Coding/Main/LocalSTTProvider.cs`
- 功能：通过HTTP调用后端STT服务
- 接口：`/api/stt/recognize`

## 后端API设计

### LLM接口（已实现）

#### POST /api/chat/
```json
{
    "message": "你好",
    "user_id": "user123",
    "session_id": "session456",
    "system_prompt": "你是一个AI助手",
    "stream": false
}
```

响应：
```json
{
    "response": "你好！有什么可以帮助你的吗？",
    "user_message": "你好",
    "session_id": "session456"
}
```

## 配置管理

### ServiceConfig
```csharp
[System.Serializable]
public class ServiceConfig
{
    public ServiceMode mode = ServiceMode.API;
    public LocalBackendConfig localConfig = new LocalBackendConfig();
    public APIConfig apiConfig = new APIConfig();
}

[System.Serializable]
public class LocalBackendConfig
{
    public string baseUrl = "http://localhost:8000";
    public string wsUrl = "ws://localhost:8000";
    public int timeout = 30;
}
```

## 优势

1. **前后端分离**：通过标准协议通信，易于维护
2. **灵活切换**：运行时切换模式，无需重新编译
3. **统一接口**：前端代码无需改动，只需切换适配器
4. **易于扩展**：新增服务只需实现接口
5. **向后兼容**：保留现有API模式，不影响现有功能

## 注意事项

1. **编码问题**：确保前后端使用UTF-8编码
2. **CORS配置**：后端需要配置允许Unity客户端访问
3. **错误处理**：统一错误响应格式
4. **超时处理**：设置合理的请求超时时间
5. **音频格式**：统一音频格式（建议WAV，24000Hz采样率）
